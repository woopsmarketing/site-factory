준비사항 :
- contabo 에서 VPS하나 구매.( 호스팅 서버 용도 )
- VPS서버내에 템플릿 복제용 도메인 하나 구매 ( 하위도메인으로 사용할것 ) - zerotheme.com
- ninper.com 이라는 사이트의 하위도메인 template1.ninper.com ~ tempalte30.ninper.com 여기에 워드프레스의 UICORE 테마로 만든 템플릿 샘플 30개가 준비되어있음.
- VPS서버안에 있는 t1.zerotheme.com ~ t30.zerotheme.com 에 해당 각 템플릿들을 복제할 예정.
- 모든 템플릿의 테마는 UICORE 이며 ELEMENTOR + ELEMENTOR PRO + ELEMEN PACK PRO + UICORE ELEMENT 와같은 플러그인으로 제작 되어있음.
- 테스트용도 도메인 1개 ( test.com )

구현할 내용 :
- VPS의 t1.zerotheme.com ~ t30.zerotheme.com 디렉토리에 모든 템플릿을 복제 ( wp-migrate 사용 경험이 있음. ) 복제할때 기존 템플릿도메인을 모두 t1.zerotheme.com 도메인으로 치환해야함.
css파일, woff 파일, 내부링크, DB등 모두 치환해야 할것같음.
- 1개의 VPS내에서 여러 도메인이 호스팅할수있도록 공유호스팅 개념으로 공간을 나누면 좋을듯함.
- 모든 도메인 및 디렉토리에는 SSL설치까지 진행한다.
- 테스트용도 도메인 1개(test.com) 도메인전용 디렉토리를 하나 만들고 해당 디렉토리에서 t1~t30.zerotheme.co.kr 템플릿중 내가 선택한 템플릿을 그대로 해당 도메인에 복제한다. 이때 복제할때에도 테스트용 도메인으로 전부 치환되어야한다.
- 현재까지 test.com 도메인에 템플릿 1개가 복제가 되어있는 상태인데, 이제 해당 템플릿에서 독립적인 사이트로 변경하기위해서 데이터를 주입해야될곳을 찾아야한다. 
- 템플릿마다 데이터를 주입해야되는곳들을 알아내는 로직이 필요하다.
- 사이트의 아이덴티티 ( 제목, 태그라인, 언어, 키워드, 사이트의 주제 등등 )
- 페이지마다 LLm을 이용해서 섹션내의 텍스트 부분에 콘텐츠 주입을 해야한다.
- 이미지도 LLM을 이용해서 제작하여 넣어줄것이다.
- 텍스트기반인 곳은 전부 콘텐츠를 주입할것이다.
- 헤더 , 푸터도 새로 콘텐츠를 주입할것이다.
- 섹션 및 페이지별로 배경색, 글자색, 글꼴의 경우에도 선택할수있으면 더더욱 독립적인 웹사이트처럼 보일것이다.
- 콘텐츠는 문맥을 벗어나지 않게 하기위해서, 하나의 공통된 사이트아이덴티티 및 프롬프트를 이용해서 전반적으로 어울리게 구성할것이다.
- 템플릿 및 페이지별로 구조를 정리하여 LLM에게 프롬프트로 전달하면 더욱 좋을것같다.
- 프롬프트의 퀄리티가 이미지 및 콘텐츠의 퀄리티를 좌지우지 할것같으니 정교하게 해야한다.

이렇게 하여 최종 완성된 전세계에 1개밖에없는 유니크한 사이트가 제작되도록할것이다.

나의 서비스를 이용하는 고객의 과정 :
1. 도메인 및 키워드 입력
2. 템플릿 30개중 선택
3. 원하는 사이트의 아이덴티티 작성 ( 텍스트기반으로 작성 -> LLM으로 해당 텍스트를 새롭게 아이덴티티 작성 )
4. 컬러 선택 ( 독립적인 배경 색 및 글꼴 및 글자 색상 선택 )

위 과정을 마무리하면 완성 된 예시 사이트를 example.zerotheme.co.kr 에 사이트를 복제하고 과정을 처리한다음 example.zerotheme.co.kr 예시 사이트를 고객에게 보여준다.
고객은 해당 예시 사이트를 보고 마음에들면 이어서 진행하기 버튼을 누르고,
네임서버 설정하면 고객 도메인과 연결된 사이트를 1개 호스팅해준다.
